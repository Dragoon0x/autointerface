# Interface Spec: Settings Panel

> A settings panel for an app that takes settings seriously. Not an afterthought buried behind
> three clicks. A well-organized, respectful settings experience.

## What is this?

A standalone settings page for a fictional note-taking app. The kind of settings panel where every option is discoverable, every toggle is clear about what it does, and changes feel immediate. Settings are usually the most neglected part of any app. This one treats them as a first-class interface.

## Who is it for?

Power users who customize their tools. The kind of person who spends 20 minutes configuring a new app before they start using it. They want control, but they also want sensible defaults. They appreciate settings that are organized well enough that they can find what they need without searching, but they also want a search bar because that's faster.

## Core sections

1. **Search.** A search input at the top that filters settings in real-time as you type. Highlights matching settings. This is the fastest way to find anything.

2. **Profile.** Name, email, avatar (placeholder), timezone. Simple form fields. Nothing surprising here.

3. **Appearance.** Theme toggle (light/dark/system), font size slider (14-22px with live preview), editor width (narrow/medium/wide), accent color picker (6 preset swatches, not a full color picker).

4. **Editor.** Toggles for: spell check, autocorrect, smart quotes, line numbers, word count, focus mode (dims everything except current paragraph). Each toggle has a one-line description beneath it explaining what it does.

5. **Sync & Storage.** Toggle for auto-sync. Dropdown for sync frequency (instant/1min/5min/manual). Storage usage bar showing "2.4 GB of 5 GB used." Export button (doesn't need to work, just styled). Clear cache button with confirmation.

6. **Keyboard Shortcuts.** A table of shortcuts. Each row: action name, current shortcut displayed as styled key caps (like ⌘+S), and an edit button. Clicking edit should show a "press new shortcut" state. Doesn't need to actually rebind, just show the interaction.

7. **Danger Zone.** At the bottom. Red-tinted section. "Delete all notes" and "Delete account" buttons. Each requires a confirmation step. This section should feel intentionally separated from the rest.

## What it should feel like

- Respectful. Every option should feel like the developer thought about whether you'd need it.
- Organized. You should be able to scan the page and find any setting within 5 seconds.
- Immediate. Toggles and sliders should show their effect instantly (at least visually). No "save" button for individual settings.
- Dense but not cluttered. More information per screen than the task manager spec, but still with clear grouping and breathing room.

## What it should NEVER feel like

- An afterthought. If it looks like a form that was generated by a framework, start over.
- Overwhelming. Even though there are many options, the grouping and hierarchy should prevent that "wall of toggles" feeling.
- Inconsistent. Every toggle, slider, dropdown, and button should feel like part of the same design system.
- Mobile-first. This is a desktop/tablet settings panel. It should work on mobile but it's designed for a wider viewport.

## Visual direction

- Clean, neutral palette. Light gray background with white cards for each section.
- Subtle borders and shadows to separate sections. Not flat, not skeuomorphic. Just enough depth.
- Monospaced text for keyboard shortcuts. Regular text for everything else.
- Form elements should feel custom but not alien. Styled toggles, sliders, and dropdowns that still feel like standard controls.

## Typography direction

- System font stack. This is a utility interface, not a marketing page.
- Clear hierarchy: section titles (16-18px, semibold), setting labels (14-16px, medium), descriptions (13-14px, regular, muted color).
- Tight but readable line-heights. This is a dense interface.

## Layout

- Left sidebar navigation listing section names. Clicking a section scrolls to it smoothly. Current section highlighted in the sidebar.
- Main content area on the right, max-width 720px.
- Sections stacked vertically with generous spacing between them.
- On mobile: sidebar collapses into a horizontal scrolling tab bar at the top.

## Interaction details

- Toggles animate smoothly (200ms ease).
- Font size slider shows a live preview text sample that resizes as you drag.
- Theme toggle immediately changes the entire settings page theme (light/dark).
- Search filters settings in real-time. Sections with no matching settings collapse. Matching text gets highlighted.
- "Danger zone" buttons show inline confirmation ("Are you sure? This cannot be undone." with Cancel/Confirm).

## Hard constraints

- Single HTML file. All CSS and JS inline.
- No external dependencies.
- All interactions must work. Toggles toggle. Sliders slide. Search filters. Theme switches.
- Dark mode must be fully implemented and switchable.
- Must work on desktop (1440px), tablet (768px), and mobile (375px).
- Page weight under 60KB.
- WCAG AA contrast in both themes.
- Keyboard navigable. Tab order should make sense. Focus states visible.
